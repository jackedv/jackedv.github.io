整理一篇程序员必备——git使用指南

¶准备工作

¶github注册

git只是一个工具，因此要想使用git上传管理自己的代码，就要有个代码管理网站，github就是在这个领域很有名的一个网站。因此首先需要去github官网注册一个账号。不过可能国内访问有时上不去，最好有个科学上网工具。

¶git安装

要想完成本地代码和互联网上github仓库的交互，就需要这么一个本地管理工具。首先去官网下载安装。

安装成功后就可以在某个文件夹右键选择Git Bash Here。之后的git命令操作都是在这进行。

¶正式使用——代码管理

¶初始化仓库

（1）在 GitHub 中新建仓库

如下图所示，点击个人主页的 Repositories 项，再点击右方的New按钮，新建一个仓库。

这里可以选择开源（public）仓库或者私人（private）仓库，一些不愿公开的个人代码备份和管理可以使用private，其他的建议使用public，发挥开源社区最大的用处。

建议勾选上“Add a README file”添加仓库简要描述文件，默认内容为Description内容。

打开新建的仓库，如下图所示，这个仓库的 Readme 文档的内容就是刚才填入的仓库名和描述说明。 我们点击Code按钮，复制 SSH 地址。

（2）准备好本地仓库内容。

本地仓库内容可以有两种来源：

  1. github已有仓库下载至本地，不需要使用仓库初始化命令
  2. 本地已有项目文件夹，包含代码、资源、数据等等，需要使用仓库初始化命令让git知道这个文件夹是一个仓库

这里只举例第一种

假设我们要下载到电脑上的目标文件夹（为了描述方便，这里简称为文件夹 A），那么我们就右键点击文件夹 A ，在右键菜单中选择“Git Bash”，在 Git Bash 客户端输入命令行：

git clone SSH地址

这样，仓库就会下载到文件夹 A 中。

当然直接 Download Zip 也是可以的。

下载仓库完成之后，文件夹 A 中有了一个新的子文件夹，比如下载了仓库 testexample2020，此时文件夹 A 中就出现了一个名为"testexample2020"的新文件夹，这就是下载到本地电脑的仓库。

这时本地仓库的文件内容就和github仓库的文件内容完全一致了，但是此时只是文件内容一致，接下来还要完成本地仓库和github仓库的绑定，并且学习每次本地仓库变动如何上传到github仓库，github仓库变动如何下载到本地仓库，这些都要用到git工具。

¶通常管理步骤

初始化仓库：

  1. 如果没有——github创建对应仓库
  2. 准备本地仓库内容
  3. 通过上传同步仓库

日常管理仓库：

  1. pull来同步仓库
  2. 在本地修改项目
  3. push上传本地仓库

¶常见使用方法

¶git基本命令

  1. git status

查看仓库的状态的命令。可以查看本地文件与本地仓库的文件区别，从而找出更改过的本地文件并加载到本地仓库。

  1. git init

初始化 Git 仓库命令，需要在项目文件夹内最外层，如”demo“项目，需要在C:xxx…/demo/ 也就是demo文件夹中使用命令

  1. git add

输入git add hit.txt命令，将hit.txt文件添加到 Git 本地缓存区


使用git status查看添加后的状态

已经添加到缓存区，等待提交到本地仓库

  1. git commit

输入git commit -m "text commit"命令，将hit.txt文件提交到 Git 仓库，并通过-m添加"text commit"的注释来标记这次提交。

  1. git push origin master & git pull origin master

push：该单词直译过来就是“推”的意思，如果我们本地的代码有了更新，为了保持本地与远程的代码同步，我们就需要把本地的代码推到远程的仓库，代码示例：

pull：该单词直译过来就是“拉”的意思，如果我们远程仓库的代码有了更新，同样为了保持本地与远程的代码同步，我们就需要把远程的代码拉到本地，代码示例：

此外，在之前我们讲到过pull request，在这里，估计大家就能更好的理解了，它表示：如果我们fork了别人的项目（或者说代码），并对其进行了修改，想要把我们的代码合并到原始项目（或者说原始代码）中，我们就需要提交一个pull request，让原作者把我们的代码拉到 ta 的项目中，至少对于 ta 来说，我们都是属于远程端的。
一般情况下，我们在push操作之前都会先进行pull操作，这样不容易造成冲突。

  1. git log

输入git log命令，打印 Git 仓库提交日志

¶分支管理

  1. git branch

输入git branch命令，查看 Git 仓库的分支情况

*号表示“当前所在的分支”。接下来输入git branch a，创建a分支

  1. git checkout

输入git checkout a命令，切换到a分支

  1. git merge

切换到master分支，然后输入git merge a命令，将a分支合并到master分支

在合并分支的时候，要考虑到两个分支是否有冲突，如果有冲突，则不能直接合并，需要先解决冲突；反之，则可以直接合并。

  1. git branch -d & git branch -D

输入git branch -d a命令，删除a分支。有的时候，通过git branch -d命令可以出现删除不了现象，例如分支a的代码没有合并到主分支等，这时如果我们一定要删除该分支，那么我们可以通过命令git branch -D进行强制删除。
